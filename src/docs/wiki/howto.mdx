**GitHubEditor**は、小さな編集を行いたいとき、また、プログラミングやGit、IDE等に馴染みの無い方向けの簡単な編集方法です。必要なのは**GitHubアカウント**のみです。

プログラミングに自信のある方は、IDEやEditorを使用して編集しましょう。

## GitHubEditor
まずはGitHubアカウントを作成しましょう。
1. [リクエストフォーム](https://forms.gle/hH44ZzWVAVspgndSA)を送りましょう。運営が確認し次第、DM致します。
2. 承認されたら、[GitHubレポジトリ](https://github.com/OyasaiServer/wiki)にアクセスしましょう。
3. src ▶ docs の順番でクリックすると、'mdx'が拡張子のファイルがいくつか見えると思います。
>>> [便利なChrome拡張機能](https://chrome.google.com/webstore/detail/atom-file-icons-web/pljfkbaipkidhmaljaaakibigbcmmpnc) (GitHubのファイルアイコンを綺麗にしてくれます)
4. 取り敢えず中身を見てみましょう。Wikiそのままのページが表示されたはずです。
5. 右上の鉛筆ボタンが見えるでしょうか？そこをクリックする事で編集モードで入ることができます。
>>> ここで編集モードに入らない人は権限を持っていません。DMを待ちましょう。
6. ファイルを編集しながら `Preview changes` からライブで確認ができます。
7. 終わったら、下の方にある `Commit Changes` よりコミットメッセージ（何をしたか）を簡単に書いてコミットしましょう。
>>> 作業が一段落ついたらこまめにコミットしましょう。Like看板みたいなものです。
8. 編集に慣れてきたら新規ページを作成してみましょう。`Add file`ボタンから作成できます。
9. コミットしたファイルは自動的にビルド（mdxからhtmlの変換）され、デプロイ(公開)されます。仮想環境にて行われるので、あなたのスペックに関係無く3分ほどかかります。しばらくしてからアクセスしてみましょう。

## IDE・Editor
※ GitHubアカウントや編集権限は取得済とします。
※ mdx はマークダウンとReactを共存させることができます。ReactComponent等も使えます。
1. コマンドラインやGitHubDesktopでレポジトリをクローンしましょう。
2. ブランチやフォークは必要ありません。mainに直接コミット・プッシュしましょう。
3. Yarnをお勧めします。`yarn start` もしくは `npm start` してから `localhost:8000` でホットリロードを使用できます。
4. `yarn install` をお忘れなく。
5. ビルドは必要ありません。PushされたらActionによって上書きされます。
6. VSCode・IntelliJ を推奨しています。
